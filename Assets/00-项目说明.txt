🎯 核心系统：通用后处理Debug系统 v2.0

特点：
✓ 自动扫描 - 发现所有Render Features
✓ 统一管理 - 一个窗口管理所有效果
✓ 零配置 - 无需手动为每个效果添加Editor
✓ AI集成 - 一键生成，Claude自动分析

打开方式：
Window → Post Process Debug Center

═══════════════════════════════════════════════════════════════════════════════

🚀 快速使用流程

1. 打开Debug窗口
   Window → Post Process Debug Center

2. 点击任意效果的 [📷 Capture Debug Data]
   • Play模式：完整效果 + 性能数据（推荐）
   • Editor模式：无需运行，但效果可能不完整

3. 对Claude说："请分析Debug数据"

4. Claude自动读取 → 分析 → 定位问题 → 修复代码

完成！

🆕 添加新后处理效果

1. 创建你的Render Feature和Shader

2. （可选）如需Debug可视化，在Settings中添加：
   public bool enableDebugVisualization = false;

3. 完成！系统会自动识别

无需任何手动配置，Debug窗口会自动显示你的效果。

💡 设计原则

1. 通用性优先
   • 所有后处理共用一套系统
   • 不为单个效果创建专用工具

2. 自动化优先
   • 自动扫描、自动识别、自动收集
   • 减少手动配置

3. 清晰的文件结构
   • 通用系统独立文件夹
   • 各效果专用文件分开管理

4. AI友好
   • 标准化输出格式
   • 一键生成完整数据包
   • Claude可直接读取分析
