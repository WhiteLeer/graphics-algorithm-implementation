==========================================
参数对比与调优 - 针对接触检测问题
==========================================

问题描述：
"无法准确找到物体接触的位置"

Unity官方默认参数：
- Intensity: 3.0
- Radius: 0.035
- Sample Count: 4（注意：Unity默认只有4个！）

我们当前默认参数：
- Intensity: 3.0 ✓
- Radius: 0.035 ✓
- Sample Count: 12 ✓（比Unity更多）

------------------------------------------
问题诊断
------------------------------------------

可能原因1：Radius太小
━━━━━━━━━━━━━━━━━━━━━━
0.035是相对于屏幕空间的比例
- 对于大场景/远距离物体：太小了
- 对于小场景/近距离物体：刚好

解决方案：
尝试增大Radius：
  0.035 → 0.05  （接触检测更好）
  0.035 → 0.1   （更明显）
  0.035 → 0.2   （最明显，但可能过度）

可能原因2：Intensity不够
━━━━━━━━━━━━━━━━━━━━━━
Intensity控制AO的可见性

解决方案：
尝试增大Intensity：
  3.0 → 4.0  （更明显）

可能原因3：Sample Count太少
━━━━━━━━━━━━━━━━━━━━━━
采样点越少，越容易漏掉接触点

解决方案：
尝试增加采样：
  12 → 16
  12 → 24（最好，但慢）

------------------------------------------
推荐测试步骤
------------------------------------------

步骤1：增大Radius测试
━━━━━━━━━━━━━━━━━━━━━━

  Intensity = 3.0
  Radius = 0.1（比默认大3倍）

  Intensity = 4.0
  Radius = 0.1
  Sample Count = 12

预期：
  ✅ AO更明显
  ⚠️ 可能过暗

步骤3：增加采样数
━━━━━━━━━━━━━━━━━━━━━━

  Intensity = 3.0
  Radius = 0.1
  Sample Count = 24

预期：
  ✅ 最平滑的效果
  ⚠️ 性能消耗更大

对比Unity官方SSAO
------------------------------------------

方法1：并排对比
━━━━━━━━━━━━━━━━━━━━━━
1. 禁用我们的SSAO
2. 启用Unity官方SSAO
3. 截图保存
4. 禁用Unity官方
5. 启用我们的SSAO
6. 调整参数直到视觉匹配

方法2：Frame Debugger对比
━━━━━━━━━━━━━━━━━━━━━━
1. 使用Unity官方，查看SSAO RT
2. 截图保存RT内容
3. 切换到我们的SSAO
4. 对比RT内容差异

关键观察点：
  - 接触位置是否都有暗影？
  - 暗影强度是否相似？
  - 边缘是否同样清晰？

------------------------------------------
场景尺度问题
------------------------------------------

重要：Radius是屏幕空间比例，不是世界空间距离！

场景类型与Radius关系：
━━━━━━━━━━━━━━━━━━━━━━
小场景（室内，相机近）：
  Radius = 0.02 - 0.05
  原因：相机近，屏幕空间1像素 = 小世界距离

中等场景（普通房间）：
  Radius = 0.05 - 0.1
  原因：标准尺度

大场景（室外，相机远）：
  Radius = 0.1 - 0.3
  原因：相机远，需要更大半径才能检测到接触

------------------------------------------
Unity官方的额外功能
------------------------------------------

我们没有实现的：
1. Downsample（降采样优化）
2. AfterOpaque模式
3. DirectLightingStrength（控制AO对光照的影响）

这些不影响接触检测，只影响性能和混合方式。

------------------------------------------
调试建议
------------------------------------------

1. 先测试Unity官方SSAO
   - 看看它能否正确检测接触
   - 如果能，记录它的参数
   - 用相同参数测试我们的

2. 使用Debug模式检查
   - Debug Mode = Depth
     确认深度有明暗变化
   - Debug Mode = Normal
     确认法线在接触处正确

3. Frame Debugger查看RT
   - SSAO_RT1应该在接触处有深色
   - 如果没有，是算法问题
   - 如果有但最终效果不明显，是混合问题
